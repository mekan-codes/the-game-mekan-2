Crane Operator: Safe Cargo Delivery - Project Structure
=======================================================

Root
----
crane_game.exe
  Built Windows executable kept in the root for a simple school-project launch
  target.

.gitignore
.gitattributes
  Repository metadata files.

Folders
-------
src/
  crane_game.c lives here.

build/
  build_crane_game.bat compiles src\crane_game.c into root crane_game.exe.

Play_Crane_Game.bat
  Root launcher. It runs crane_game.exe with runtime\ added to PATH.

assets/
  backgrounds/       Background image used by the game.
  world/             Crane, cargo, beams, delivery dock, gate, fan, and world sprites.
  effects/           Lightweight effect sprites.
  ui/                Menu and HUD UI sprites.
  hazards/           Static hazard sprites.
  animations/        Optional SDL2_image animation frame folders.
  fonts/             Font fallback used by load_font_size().

runtime/
  SDL2 and related runtime DLL fallbacks. These are not left in the root folder.

docs/
  README.txt and this project-structure note.

Build/Run Flow
--------------
From the repository root:

build\build_crane_game.bat
Play_Crane_Game.bat

Why The Launcher Exists
-----------------------
crane_game.exe is dynamically linked against SDL2/SDL2_ttf and optionally
SDL2_image. Windows normally searches the executable directory and PATH for
those DLLs; it does not automatically search runtime\. Play_Crane_Game.bat
temporarily adds runtime\ to PATH, preserving a clean root folder without
pretending that root-only double-click is guaranteed on every Windows machine.

Removed Reference Material
--------------------------
Old assignment folders, old asset-pack folders, and one-off patch scripts were
removed after their required assets, font, and runtime DLLs were copied into
assets/ and runtime/.
